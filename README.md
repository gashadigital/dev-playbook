# Gasha Digital Dev Playbook

A curated collection of technical workflows, manual setup guides, and best practices for Full-Stack and Mobile development. This playbook is designed to standardize development environments and streamline the onboarding process for new projects.

---

## 📖 Table of Contents

### 📑 Chapter 1: Version Control

* **[1.1 Introduction to Git](docs/git_basics.md)**: Essential concepts and setup.
* **[1.2 Daily Workflow](docs/git_workflow.md)**: Staging, committing, and pushing.
* **[1.3 Branching & Teamwork](docs/teamwork.md)**: Managing features and updates.

## 🛠️ Getting Started

To get the most out of these documents, it is recommended to follow the chapters in order. Starting with **Chapter 1 (Git)** ensures that all your subsequent environment configurations are versioned and tracked correctly.

## 🤝 Contribution

When adding new documentation to this playbook:
